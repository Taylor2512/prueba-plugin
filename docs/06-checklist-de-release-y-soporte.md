# Checklist de release y soporte

## Antes de publicar

- [ ] Surface API congelada
- [ ] Tipos públicos revisados
- [ ] Build ESM/CJS funcionando
- [ ] `.d.ts` generados
- [ ] CSS del editor exportado
- [ ] Examples funcionando
- [ ] Tests unitarios verdes
- [ ] Playwright crítico verde
- [ ] Changelog actualizado
- [ ] Licencia y README por paquete listos

## Antes de vender a terceros

- [ ] Branding sin referencias heredadas innecesarias
- [ ] Contratos de persistencia/API/form-json documentados
- [ ] Política de soporte definida
- [ ] Roadmap de compatibilidad
- [ ] Modo enterprise claramente separado
- [ ] Paquetes experimentales etiquetados

## Soporte recomendado

### Nivel 1
- integración básica
- instalación
- ejemplos

### Nivel 2
- custom fields
- theme/branding
- persistencia/API/form-json

### Nivel 3
- colaboración
- performance
- white-label
- debugging de generación/conversión

## KPIs útiles

- tiempo de integración inicial
- bundle size del editor
- tiempo de render de templates grandes
- tiempo de generate/converter
- cantidad de breaking changes por release
- adopción por paquete

## Cierre

La sexta tanda completa la transición documental desde un repositorio potente pero interno hacia una plataforma empaquetable y distribuible.
